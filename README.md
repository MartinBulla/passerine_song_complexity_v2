## Supporting information for 'Global patterns and evolution of avian song complexity'

Contains data, R code, and resources used to reproduce the analyses and outputs for [Bulla et al. (2026)](ADD-LINK-TO-PREPRINT).

When referring to or reusing these materials, **please cite** our [Bulla et al. (2026) Link to be ADDED](ADD-LINK-TO-PREPRINT) and this repository<sup>[1](#1)</sup>.

***

### **Repository contents**
[**HTML Supporting information**, including code](https://martinbulla.github.io/passerine_song_complexity_v2/) is generated from the following repository structure:

[Data](Data/) stores:
- *raw data*:
  - [`song_raw.csv`](Data/song_raw.csv) contains raw data for individual song recordings.
  - [`song_complexity_repeatability.txt`](Data/song_complexity_repeatability.txt) contains complexity scores from 8 observers, used to estimate between-observer repeatability (see [`DAT_9_repeatability.R`](R/DAT_9_repeatability.R))
- *data from published sources*: files indicated with `_` prefix; file names indicate the source (see the manuscript for detail and exact citations and original sources for data description)
- *manipulated data*: files with `DAT_` prefix generated by our R scripts (having also `DAT_` prefix).
- *sqlite data*: spatial databases  [`DAT_rangeMapper.sqlite`](https://osf.io/nf6vc/files/8c46r) and [`DAT_rangeMapper_all.sqlite`](https://osf.io/nf6vc/files/5zcrh)  are necessary for spatial analyses, but due to their large size are stored at [OSF](https://osf.io/nf6vc/). For the scripts to run, download these files and place them in the project's `Data/` directory.
- [*Birds of the World illustrations*](Data/BoW_illustrations/) used in Figure 2B.

[R](R/) stores scripts used in the analysis:
- [`_runRmarkdown.R`](R/_runRmarkdown.R) generates the HTML [Supporting information](https://martinbulla.github.io/passerine_song_complexity_v2/) from [`MS_online_material.R`](R/MS_online_material.R).
- [`MS_online_material.R`](R/MS_online_material.R) is the script behind the HTML [Supporting information](https://martinbulla.github.io/passerine_song_complexity_v2/), containing all code used to generate the paper outputs.
- `DAT_` prefix files prepare data for other scripts and analyses (script-specific information is provided in each script's header).
- [`__init__.R`](R/__init__.R) contains functions and constants loaded by other scripts.

[Output](Output/) stores separate files of all outputs used in the manuscript:
- [HTML of the Supporting information](Output/SI.html).
- SVG of Inkscape polished Fig. 2.
- CSVs of the tables.
- [model assumptions](Output/modelAss) folder contains diagnostic outputs for all fitted models.
- [model outputs](Output/est_out) folder stores model output files for reproducible reporting, i.e. table generation.

[Resources](Resources/) stores
- [_bib.bib](Resources/_bib.bib) bibliography used in the HTML [Supporting information](https://martinbulla.github.io/passerine_song_complexity_v2/).
- [styles.css](Resources/styles.css) define graphical parameters for the HTML generation.
